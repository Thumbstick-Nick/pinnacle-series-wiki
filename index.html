Peak Experience vs Classic

Pinnacle Series ships in two UIs. Peak Experience is the redesigned modern interface and the default for new tenants. Classic is the legacy UI that some tenants are still on while they migrate. Tenant-level toggles split for admin and end-user UIs let customers stage the rollout.

Important visual distinction. Classic Pinnacle Series uses a thin blue header bar with icon-only buttons. The yellow-headered UI you may have seen in old marketing material is the KnowledgeSmart admin dashboard — a completely different product. Don't conflate the two when demoing or comparing.

Classic UI — what users actually see

For tenants still on Classic, the end-user Home page looks like this:

  • Top: thin blue navigation bar with icon-only buttons (right-aligned): home, briefcase, graduation cap, trophy, bell, chat, gear, user avatar.
  • Left sidebar: Asset Libraries — flat hierarchical list of every library the user can access (Adobe, Autodesk, Autodesk Forma, Bentley, Bluebeam, Microsoft, Procore, Revizto, Solibri, SOLIDWORKS, etc.). Libraries with sub-content show a + to expand.
  • Center column: Search bar at top; body has MY WORK GROUPS, MY COURSES (with progress bars), and TRENDING ASSETS sections.
  • Right column (stacked): User profile card · Upcoming Live Events panel · My Assets · Download Add-ons with Pinnacle Management Utility, Pinnacle User Tools, and a Deployment Guide link.
  • Bottom-right: floating Support button (blue).

What disappears in Peak: the asset-library tree on the left, MY WORK GROUPS, TRENDING ASSETS, and Upcoming Live Events on the Home page. Peak's Home is Assignments + Libraries + (optional) SPARK.

The 2-step email-first sign-in

The Pinnacle Series sign-in page is email-first. The flow is:

  1. Page opens with the Email field, a Set Your Language selector (default: English), and a green Continue button.
  2. User enters their email and clicks Continue.
  3. The page updates: if the user signs in with a password, a Password field, Remember this device checkbox, and Forgot password? link appear. User enters password and clicks Sign In.
  4. User lands on Home.
Default-tenant caching. The browser may cache a default tenant in local storage from a previous session. A user signing in for the first time on a device may see different behavior than a returning user. Clearing local storage / using a private window resets this.

Single Sign-On (SSO)

SSO is tenant-scoped and user-scoped — both have to line up for a user to be routed through SSO instead of password sign-in.

  • Tenant prerequisite: SSO integration must be configured for the tenant via Admin Portal → Security & Authentication.
  • Per-user override: Each user has an "Exclude User from SSO" toggle (Admin Portal → Users → Edit User Details → User Settings). When ON, that user signs in with a password even if their tenant has SSO.

When an SSO-eligible user enters their email and clicks Continue:

  • The Password field is not shown.
  • If the user already has a valid IdP session, they're taken directly into Pinnacle Series.
  • Otherwise, they're redirected to the Identity Provider to authenticate, then returned to Pinnacle Series.

Naviate (custom SSO for a specific customer)

Naviate is NOT a typo. Naviate is a named customer (a Symetri subsidiary) for whom we built a custom SSO integration. Naviate users authenticate via My.Symetri.com, not the Pinnacle password field.

  • URLs that include a NaviateID=<email> query parameter route the user through the Naviate SSO flow.
  • If the user is logged into My.Symetri.com, Pinnacle authenticates the user via Naviate. If no Pinnacle account exists, one is created on the fly with a Terms & Conditions modal before landing.
  • If not logged into My.Symetri.com, the user is sent to the Naviate sign-in page first.
  • If the user has no valid Naviate account, no Pinnacle account is created and the user is not signed in.
  • URLs without NaviateID follow the standard Pinnacle sign-in flow.
  • Naviate integration is production-only — test environments may have limited coverage.
SSO configuration permission: Configuring SSO for a tenant requires the Administrator permission (under User Administration). Configuring an individual user's "Exclude from SSO" toggle requires the same permission.

AI Chat

Where Home page → left nav → AI Chat (only when the tenant has AI Chat enabled in Admin Portal → Settings → General)
Correction: AI Chat is reached via the AI Chat top-bar button (not a left-nav item) when the tenant has Allow Users to Access AI Chat (Preview) enabled. It opens a dedicated page titled "AI Chat" with the assistant persona named "Pinnacle AI".

Conversation experience

  • Persona name: "Pinnacle AI" (alongside the user's "You" turn).
  • Typing indicator: dots while the model is generating.
  • Numbered citations (¹ ² ³ …) tie each assertion back to a specific Pinnacle Series content item.
  • Clickable content cards at the bottom of the reply — each card shows the content title, content type (Video / Document / etc.), and a small icon. Clicking a card navigates straight to the source.
  • Disclaimer line: "AI-generated content may be incorrect."
  • Page footer: dark New Chat button (resets conversation) + "Ask a question" text input + send button (paper-plane icon).

AI Chat is a context-aware chat feature that helps users find answers and content within Pinnacle Series using natural language. It's intended for in-the-moment troubleshooting, learning suggestions, and quick references — not for replacing structured learning paths.

RAG Retrieval-Augmented Generation grounds every answer in the customer's own Pinnacle Series content. The model is shared across tenants but content access is strictly isolated — AI Chat will never surface content from another customer's library.

The technology stack

Microsoft Azure OpenAI

Built on Azure OpenAI Service using the GPT-3.5 Turbo model. Microsoft does not use customer data to train or improve the model.

RAG-grounded answers

Each response is drawn from Pinnacle Series content the user has explicit access to, with numbered citation markers and clickable source cards.

Per-tenant isolation

Content access is strictly scoped per tenant. AI Chat respects the same library-sharing rules as Search and Browse.

SPARK section on the Home page

1–2/day SPARK is designed for "1–2 SPARKs a day" — a sustainable pace that builds skill over time without overwhelming busy schedules. Up to 25 videos queued per session, each 3–5 minutes.

How users experience SPARK

First-time setup

On first sign-in after SPARK is enabled for their group, users see a welcome screen with optional preference capture: Industries, Topics & Interests, and a free-text "Gist for you" field. Skippable.

Daily feed

Up to 25 videos in a session queue, each 3–5 minutes. Designed to fit between focused work blocks.

Progress tracking

Completed videos get a green check; the queue refreshes as videos are completed so the feed stays fresh.

Save & share

Bookmark a SPARK for later or share its link with a colleague.

How SPARK picks content

SPARK uses a weighted match across three signal sources, in priority order:

PRIORITY 1

User preferences

The user's own "Just for You" settings. Highest weight; overrides everything else when set.

PRIORITY 2

Group interests

Topics chosen by an admin at the group level. Used when individual prefs aren't set or to supplement them.

Hero Products

SCORM Export Admin Portal New feature

Where Admin Portal → Content / Learning Paths → Export as SCORM (on eligible learning paths or courses)

SCORM Export lets a customer package selected Pinnacle Series learning content into SCORM files for upload to their own LMS (Cornerstone, Workday Learning, a corporate LMS, etc.). Learners launch the activity from inside that external LMS, but their progress, completions, and scores are reported back to Pinnacle Series.

This is new functionality. Detailed UI flow and test coverage are still being developed. Behavior captured here may evolve. Check Pinnacle_UI_Facts.md §6a for the most current capture.

UI flow (per per-item kebab)

SCORM Export is initiated from a per-item kebab on a content list (typically a Library Details page). Two kebab options on a Learning Path item:

  • Export Learning Path to SCORM — exports the entire LP as one package.
  • Export Course to SCORM — opens the Select Course to Export modal.

Select Course to Export modal

  • Title: Select Course to Export.
  • Search bar: "Search Course".
  • Columns: Course Name (sortable), Publisher, Modified.
  • Each course in the LP is row-selectable.
  • Footer: Cancel + green Export Course button (singular — one course per export action).

Validating SCORM packages externally

QA validates SCORM packages by uploading them to SCORM Cloud — the standard external playback target for any SCORM-export test case. This confirms the package itself is valid SCORM regardless of the Pinnacle-side gating documented above.

What can be exported

  • Learning Paths — only a limited subset of learning paths is eligible for SCORM export. Not every learning path will have the option.
  • Courses within a learning path — when exporting, the admin can choose to export the entire learning path OR specific course(s) from within it.

How it works for the learner

  1. An admin exports an eligible learning path (or selected courses) as a SCORM package from Pinnacle Series.
  2. The customer uploads the SCORM package into their own LMS.
  3. A learner launches the SCORM activity from inside that LMS — they do not sign in to Pinnacle Series directly.
  4. The learner consumes the content. Progress and completion are reported back to Pinnacle Series via the SCORM package's link to the issuing tenant.

Prerequisites for the learner to consume the content

All four conditions must be true at the moment the learner launches the SCORM activity, not just at export time:

  1. The learner has an active Pinnacle Series account in the same tenant the SCORM package was exported from.
  2. The learner's email in the external LMS matches their email in Pinnacle Series — email is the identity correlation key.
  3. The learner's Pinnacle Series permissions and content visibility still allow access to the underlying content (libraries shared with their group, content not deleted, etc.).
  4. The tenant's Pinnacle Series subscription is active.

What blocks access

If any of the conditions above fails, the SCORM activity will not launch successfully or will not report progress back. Common blocking conditions:

  • Pinnacle Series subscription has expired.
  • The learner does not have a Pinnacle Series account in the tenant the package was exported from. (E.g., a vendor or external partner who is not in the tenant cannot consume the package.)
  • The learner's email in the external LMS does not match their email in Pinnacle Series.
  • The underlying content is no longer accessible to the learner in Pinnacle Series — library not shared with their group, content unpublished, or content deleted.
What CSM / Support should ask first when learners report broken SCORM activities:
  1. Is the learner's email the same in both the external LMS and Pinnacle Series?
  2. Is the learner still active in Pinnacle Series in the tenant the package was exported from?
  3. Does the learner still have access to that specific content in Pinnacle Series (library sharing intact, content still published)?
  4. Is the tenant's subscription still active?
Common Sales question: "Can a vendor or external partner who isn't in our Pinnacle tenant consume our SCORM package?" Answer: No. The package is bound to the issuing tenant. Each learner consuming the SCORM package must have an active account in that same tenant.
Required permission to export SCORM: Likely Content Administrator or Content Publisher, but this has not been visually confirmed yet. Will be updated as test cases are authored.

Common Questions

What permissions does someone need to add users to Pinnacle Series?

Administrator (under User Administration). This grants the ability to add users manually, import them via spreadsheet, sync them from Entra ID / Active Directory, and assign them to groups. See Users and Permissions Reference.

What permissions are needed to publish content?

Content Administrator (full) or Content Publisher (limited to libraries shared with the publisher). A Content Author can create content but cannot publish it independently.

What permissions are needed to view reports?

Reporting Administrator for all users, or Reporting Manager for users the person manages or groups they own. (Older docs may call this "View Reports" — that name has been renamed.)

How do I get to the Recycle Bin?

Admin Portal → Content → Tools Menu (kebab icon on the toolbar) → Recycle Bin. It is not a left-nav item.

Where do I configure SSO?

Admin Portal → Security & Authentication (top-level item, not under Settings). Then scroll to the Single Sign-On Authentication section.

Why doesn't a user see SPARK?

SPARK is enabled per group. If the user's group does not have SPARK toggled on, they won't see SPARK in the left nav or on the Home page. Check Admin Portal → Groups → select the group → Edit Group → SPARK toggle. Also note: if no relevant libraries are shared with the group, the SPARK feed may appear empty.

Why does an Admin Portal user see fewer nav items than expected?

The Admin Portal left nav is permission-filtered. Each user only sees the sections their permissions grant access to. If a user has only Content Administrator, they'll see Admin Home, Content, and Libraries — and nothing else. Granting additional permissions (e.g., Administrator for Users, Assignment Administrator for Assignments) expands what they see.

Why is "Naviate" spelled that way?

Naviate is the name of a specific customer (a Symetri subsidiary) for whom Eagle Point built a custom SSO integration. It is not a typo of "Navigate." See Sign In & Authentication for the Naviate auth flow.

What's the difference between Peak and Classic?

Peak Experience is the modern UI; Classic is the legacy UI. Tenants migrate at their own pace. Peak is enabled by two independent tenant toggles: Peak Experience Configuration (admin UI) and Peak Experience for End Users (Home page). See Peak vs Classic.

Does AI Chat use customer data to train the model?

No. Microsoft Azure OpenAI does not use customer data to train or improve the model. Inputs and outputs are not retained. AI Chat uses RAG to pull from content the user has explicit access to. See AI Chat.

Where do users download Pinnacle User Tools?

Top-right user avatar → Profile & Settings → Account Info tab → Download Add-Ons section. The page lists both Pinnacle Management Utility and Pinnacle User Tools. Visibility of these download links is controlled by the Administrator permission on the tenant.

Why did the sign-in page only show an email field at first?

The sign-in page is email-first. Step 1 shows only Email + Set Your Language + Continue. After clicking Continue, step 2 reveals Password + Remember this device + Forgot password? — unless the user is SSO-eligible, in which case the password field is never shown and they're routed to their identity provider. See Sign In & Authentication.

What's the difference between an Assignor and an Assignment Administrator?

Both can create assignments. The difference is scope: Assignment Administrator manages assignments and external learning for all users in the tenant. Assignor is scoped to users they manage, users in groups they own (or child groups, if Inherit Permissions is on), and assessment assignments to interviewees. See Permissions Reference.

What's an Interviewee?

A KnowledgeSmart user-status used when an admin assigns an assessment to someone who isn't yet a user in the tenant — typically a job candidate. The interviewee gets an emailed assessment link and does not need a Pinnacle or KS account to take it. If they later become a tenant user, the KS status flips from Interviewee to Employee (the KS account persists). See Glossary.

Where do I configure password complexity, expiry, or account lockout?

Admin Portal → Security & Authentication → Password Authentication. Four controls: Use Complex Passwords, Password Expires in N Days (default 90), Cannot Match Previous N Passwords (default 10), Lock Out User After N Invalid Attempts (default 10).

Can I assign content to internal and external users in one operation?

Yes. The Add Users to Assignment modal in Assignment Management shows both Standard and External users in the same picker, with separate selection counts. The User Type column distinguishes them with badges.

Why doesn't a user see the MANAGEMENT section in their left nav?

The MANAGEMENT section is rendered only for users with at least one admin-tier permission. Users without any Admin-Portal-granting permission see no MANAGEMENT section at all — no placeholder. Granting any admin permission (Content Administrator, Assignment Administrator, Administrator, etc.) makes it appear.

How do tenants control which features are exposed to end users?

Tenant-wide toggles live at Admin Portal → Settings → General. Notable ones: Allow Users to Access AI Chat (Preview) gates the AI Chat top-bar button; Allow Users to Submit External Learning gates the matching top-bar button and avatar dropdown item; Allow Users to Download Add-ons gates the Pinnacle Management Utility and User Tools download links in My Account.

What's the difference between Custom and Standard themes?

Standard themes are Eagle Point-provided defaults like "Autumn Bliss" — available out of the box. Custom themes are admin-created with three independently-configurable color slots: Navigation, Primary, Secondary. Multiple themes can be saved per tenant; the active one shows a green CURRENT THEME badge. See Theme Customization.

How does Active Directory sync work?

Three mutually-exclusive options at Admin Portal → Integrations → Active Directory: Do Not Sync, On-Premises Active Directory (configured via the legacy Pinnacle Management Utility), or Microsoft Entra ID (configured directly in the Peak admin portal with four sub-configs: Group Sync Options, Rule Sets, Map User Attributes, and Sync Logs with a 30-day window).

What happens to a SCORM package if the customer's Pinnacle subscription expires?

The SCORM activity stops launching successfully and progress no longer reports back. Active Pinnacle Series subscription is one of four prerequisites checked at consumption time (not just at export time). The other three: learner has an active Pinnacle account in the issuing tenant, email matches between the external LMS and Pinnacle, and the learner still has access to the underlying content. See SCORM Export.

Can a vendor or external partner consume a SCORM package we exported?

No. The package is bound to the issuing tenant. Each learner consuming the SCORM activity must have an active account in that same Pinnacle Series tenant. There's no way around this — by design.

How is SPARK content selected for a user?

SPARK uses a weighted match across three signals, in priority order: (1) the user's own "Just for You" preferences, (2) topics chosen by an admin at the group level, (3) Hero Products fallback (AutoCAD, Civil 3D, Revit, etc.). Conditions: not viewed in the last 90 days, under 5 minutes long, and the match phrase must appear in the content's Title, Description, Keywords, Topics, Chapters, or Transcript. See SPARK.

What's the difference between the Username and the user's full name?

The Username is editable free text — it drives the Welcome heading on Home (Welcome, [Username]) and the prominent name on My Account. The display/full name is sourced from the user record and shown as a secondary line below the email. They're independent — a user can change their Username without touching their full name.

Is Naviate a typo of "Navigate"?

No. Naviate is a specific customer (a Symetri subsidiary) for whom Eagle Point built a custom SSO integration. URLs with a NaviateID=<email> parameter route the user through Naviate's My.Symetri.com auth instead of Pinnacle's password flow. See Sign In & Authentication.

How long do items stay in the Recycle Bin?

Indefinitely — there's no automatic expiry. Captured tenants show items 14+ months old. To restore: select rows → click the Recycle icon top-right (or use the per-row kebab → Restore to Previous Location). To purge permanently: Trash icon top-right or per-row kebab → Delete. See Content.

Why is "View Reports" called something different now?

It was renamed to Reporting Administrator in the current Pinnacle UI. A new scoped variant Reporting Manager was also introduced, scoped to users the person manages or groups they own. Older training material and the Role-Based Permissions Guide PDF still use the old "View Reports" label — that name is deprecated. See Permissions Reference.

How do I check the status of an Entra ID sync?

Admin Portal → Integrations → with Microsoft Entra ID selected, click the Microsoft Entra ID Sync Logs "Logs" button. Shows the last 30 days of sync activity.

What does "Group Type" mean when I create a group?

Pinnacle Series groups are typed at creation. The Add Group Manually modal has a Group Type dropdown with two values: General (default) and Job Role. Behavioral differences across assignments, SPARK, content sharing, and reports aren't yet fully documented.
