Pinnacle Series Wiki

A shared reference for how Pinnacle Series actually behaves — what's where, who can do what, and the small details that trip people up.

Welcome / Overview

Pinnacle Series is Eagle Point Software's learning and productivity platform. It combines a structured library of curated learning content with on-demand search, AI chat, and personalized skill development tools. It's used by AEC and manufacturing organizations to onboard new hires, support continuing education, and answer in-the-moment "how do I do this?" questions.

Three ways users find what they need

  • Global Search. A keyword or phrase typed into the top-right search bar returns videos, guides, workflows, and documents — filtered to what the user has access to.
  • Library Browsing. Structured content organized by discipline, product, or job function. Good for discovering new topics or working through a learning path.
  • AI Chat. Natural-language Q&A backed by Pinnacle's content. Pulls only from content the user is authorized to see. More on AI Chat →

Two experiences: Peak and Classic

Pinnacle Series ships in two UIs — Peak Experience (the modern UI, default for new tenants) and Classic (the legacy UI, still in use at some tenants mid-migration). All new feature work targets Peak. More on Peak vs Classic →

Who uses this wiki: Sales, CSM, Support, QA, new hires. The goal is a single trustworthy reference for "what does this do, and what permissions does it need." If you find something wrong or missing, flag it to Nick.

Peak Experience vs Classic

Peak Experience is the redesigned Pinnacle Series UI. Classic is the older interface that some tenants are still on while they migrate. The two experiences are controlled by tenant-level toggles — and importantly, those toggles are split for admin and end-user UIs.

The two Peak Experience toggles

Peak Experience is controlled by two independent tenant-level toggles at Administration Portal → Settings → General → Peak Experience:

ToggleControls
Peak Experience ConfigurationEnables Peak for the Administration Portal (admin-side UI).
Peak Experience for End UsersEnables Peak for the Home page / end-user experience.

The two are intentionally independent so a tenant can configure Peak in the admin portal first — at their own pace — before flipping the end-user experience.

Why move to Peak

  • Modern UI. Streamlined nav, bulk modification tools, enhanced search.
  • Library-based content management. Libraries, Topics, and Subtopics replace the old folder-based structure.
  • Library-driven permissions. Access is now controlled at the library level rather than per-folder, with less manual effort.
  • Centralized assignments. Both skills assessments and learning tasks managed in one place; strong visual markers for due dates and progress.
  • Future-proofed. Peak is the foundation for AI-driven assistance, smarter reporting, and continuous feature development.

What's not yet in Peak

  • Live Events — not yet available.
  • Workgroups — still in development. Note the related Work Group Management permissions exist but apply only to Classic.
  • Document conversion for search & translation — converting external docs into Pinnacle Documents for enhanced searchability and translation isn't yet supported.

Sign In & Authentication

Where https://portal.pinnacleseries.com/ (or tenant-specific URL)

The 2-step email-first sign-in

The Pinnacle Series sign-in page is email-first. The flow is:

  1. Page opens with the Email field, a Set Your Language selector (default: English), and a green Continue button.
  2. User enters their email and clicks Continue.
  3. The page updates: if the user signs in with a password, a Password field, Remember this device checkbox, and Forgot password? link appear. User enters password and clicks Sign In.
  4. User lands on Home.
Default-tenant caching. The browser may cache a default tenant in local storage from a previous session. A user signing in for the first time on a device may see different behavior than a returning user. Clearing local storage / using a private window resets this.

Single Sign-On (SSO)

SSO is tenant-scoped and user-scoped — both have to line up for a user to be routed through SSO instead of password sign-in.

  • Tenant prerequisite: SSO integration must be configured for the tenant via Admin Portal → Security & Authentication.
  • Per-user override: Each user has an "Exclude User from SSO" toggle (Admin Portal → Users → Edit User Details → User Settings). When ON, that user signs in with a password even if their tenant has SSO.

When an SSO-eligible user enters their email and clicks Continue:

  • The Password field is not shown.
  • If the user already has a valid IdP session, they're taken directly into Pinnacle Series.
  • Otherwise, they're redirected to the Identity Provider to authenticate, then returned to Pinnacle Series.

Naviate (custom SSO for a specific customer)

Naviate is NOT a typo. Naviate is a named customer (a Symetri subsidiary) for whom we built a custom SSO integration. Naviate users authenticate via My.Symetri.com, not the Pinnacle password field.

  • URLs that include a NaviateID=<email> query parameter route the user through the Naviate SSO flow.
  • If the user is logged into My.Symetri.com, Pinnacle authenticates the user via Naviate. If no Pinnacle account exists, one is created on the fly with a Terms & Conditions modal before landing.
  • If not logged into My.Symetri.com, the user is sent to the Naviate sign-in page first.
  • If the user has no valid Naviate account, no Pinnacle account is created and the user is not signed in.
  • URLs without NaviateID follow the standard Pinnacle sign-in flow.
  • Naviate integration is production-only — test environments may have limited coverage.
SSO configuration permission: Configuring SSO for a tenant requires the Administrator permission (under User Administration). Configuring an individual user's "Exclude from SSO" toggle requires the same permission.

Home Page (End User)

Where Default landing page after sign-in.

Left navigation

The end-user left nav, in order:

  1. Home
  2. Assignments
  3. Libraries
  4. Bookmarks
  5. SPARK — only if the user belongs to a group with SPARK enabled.
  6. AI Chat
  7. MANAGEMENT section (containing Administration Portal) — only if the user has any admin-tier permission.
MANAGEMENT section behavior. The MANAGEMENT section is rendered only for users whose permissions grant access to the Administration Portal. It is expanded by default on every fresh Home page load (the user can collapse it, but it re-expands next time). End users without any Admin-Portal-granting permission see no placeholder — the section is simply absent.

Page body

The Home page body shows:

  • Assignments section — the user's currently assigned content.
  • SPARK section (if SPARK is enabled for the user's group) — header reads "SPARK — AI-powered learning bursts" with an "Up Next" panel and a playlist of short video cards.
  • Featured Libraries — appears only if at least one library is flagged as Featured in Admin Portal → Libraries. Absent entirely otherwise.
  • Libraries — always present. The non-featured carousel of all libraries shared with the user.

Top bar

  • Global Search (top-right).
  • User avatar (top-right) → menu opens with Profile & Settings, Switch Tenant, Sign Out.

Assignments (End User)

Where Home page → left nav → Assignments

The end-user Assignments page shows the user's currently assigned learning content with strong visual markers for due dates, progress, and completion status.

What's there

  • Currently Assigned. Active assignments, sortable by due date.
  • Enrollment History. All completed, in-progress, and dropped courses.
  • Per-row kebab menu with quick actions: View details, Re-Enroll, View Certificate, Drop (where allowed).

Common end-user actions

  • Self-enroll. User can browse libraries and enroll themselves in a course or learning path without admin involvement.
  • Re-Enroll. Take a course again — useful after content updates or for re-certification.
  • View Certificate. Available for completed enrollments.
  • Pause Learning. Stop progress temporarily and resume later.
Required permissions: None. Every user has access to their own Assignments page.

Libraries (End User)

Where Home page → left nav → Libraries

The Libraries page shows all libraries the user has been granted access to. Users only see libraries that an admin has shared with them, their group, or all users.

Library views

  • Library cards grouped by visibility (Featured, Subscription, Custom).
  • Per-library: Topics button to jump to a topic-filtered view.
  • Library Search Bar (top-left of a library page) for search scoped to that library.

Inside a library

  • Filter panel with Publisher, Topic, Sub-topic, Content Type, Duration, Difficulty.
  • Content cards for videos, documents, workflows, courses, learning paths, and quizzes.
  • Per-content actions: Like, Share, Bookmark, Send Feedback.
Required permissions: None. Library access is controlled by the admin via library sharing — once shared, no special permission is needed to view.

Bookmarks

Where Home page → left nav → Bookmarks

The Bookmarks page lists every piece of content the user has bookmarked. Bookmarking is the user's way of saving a reference for quick access later.

  • Each piece of content has an "Add to Bookmarks" action available from its detail view or kebab menu.
  • Bookmarks persist across sessions and devices for that user.

AI Chat

Where Home page → left nav → AI Chat (only when the tenant has AI Chat enabled in Admin Portal → Settings → General)

AI Chat is a context-aware chat feature that helps users find answers and content within Pinnacle Series using natural language. It's intended for in-the-moment troubleshooting, learning suggestions, and quick references — not for replacing structured learning paths.

How it works

  • Built on Microsoft Azure OpenAI Service, using the GPT-3.5 Turbo model.
  • Uses Retrieval-Augmented Generation (RAG) — answers are drawn from Pinnacle Series content the user has explicit access to.
  • The model is shared across all tenants, but content access is strictly isolated per tenant. AI Chat will never surface content from another customer's library.

Tips for users

  • Be specific with queries.
  • Add content-type words like "video," "guide," or "tutorial" to narrow what gets returned.
  • Use Boolean operators (AND, OR, NOT) for refinement.
  • Click "New Chat" to reset conversation context.

Data privacy

AspectHow AI Chat handles it
What gets sentOnly user queries. AI Chat is not designed to handle sensitive personal or commercial information.
Data retentionInputs and outputs are not retained.
Model trainingMicrosoft Azure OpenAI does not use customer data to train or improve the model. Customer inputs are isolated from Microsoft and OpenAI.
ComplianceCCPA, GDPR, SOC 2, SOC 3, CSA Star.
Required permissions for end users: None — once enabled at the tenant level, AI Chat is available to all users in the tenant.
Required permissions to enable AI Chat for a tenant: Administrator (toggle is in Admin Portal → Settings → General).

SPARK

Where Home page (left nav and body section) — only if the user is in a group with SPARK enabled.

SPARK is Pinnacle Series' personalized nano-learning feed. It curates short videos (under 5 minutes) from libraries the user can access, prioritizing content that matches the user's interests and group's focus areas.

How users experience SPARK

  • First-time setup. On first sign-in after SPARK is enabled for their group, users see a welcome screen with optional preference capture: Industries (multi-select), Topics & Interests (software tools), and a free-text "Gist for you" field. Skippable.
  • Daily feed. Up to 25 videos in a session queue, each 3–5 minutes. The intent is "1–2 SPARKs a day."
  • Progress tracking. Completed videos get a green check; the queue refreshes as videos are completed.
  • Save & share. Bookmark a SPARK or share its link with a colleague.

How SPARK picks content

SPARK uses a weighted match across:

  1. User preferences (highest priority — the user's "Just for You" settings)
  2. Group interests (set by an admin at the group level)
  3. Hero Products from shared libraries — fallback when no preferences are set: AutoCAD, AutoCAD Civil 3D, InfraWorks, Revit (Architecture / MEP / Structure), Bluebeam, Inventor, Navisworks, Autodesk Construction Cloud.

Match conditions:

  • Content has not been viewed by the user in the last 90 days.
  • Content is under 5 minutes.
  • The match phrase appears in Title, Description, Keywords, Topics, Chapters, or Transcript.

Configuring SPARK (admin side)

  1. Define interests: Admin Portal → Settings → Users → User Interests area → Add Interest.
  2. Enable per group: Admin Portal → Groups → select group → Edit Group → SPARK toggle (top right) → Enable.
  3. Set notification cadence: Daily / Weekly / Monthly. Weekly is the default recommendation for most teams.
  4. Choose group interests: select from the User Interests catalog the topics most relevant to this group.
Best practice for group interests. Plan around workflows and deliverables, not departments or job titles. "Drainage Modeling" produces better SPARK matches than "Civil Team."
Required permissions:
  • Defining User Interests / enabling SPARK at the group level: Administrator (User Administration).
  • End users seeing SPARK: must be a member of a group where SPARK has been toggled on.

My Account / Profile & Settings

Where Top-right user avatar → Profile & Settings
Naming note: The menu item reads Profile & Settings, but the destination page header reads My Account. They are the same page.

The user avatar dropdown

Clicking the user avatar in the top-right of the end-user UI opens a small menu:

  • Profile & Settings — always shown. Opens the My Account page.
  • Submit External Learning — conditionally shown. Only appears when the tenant has "Allow Users to Submit External Learning" toggled on at Admin Portal → Settings → General. If the toggle is off, this menu item is absent for every end user in the tenant.
  • Switch Tenant — shown for users who have access to more than one tenant.
  • Sign Out — always shown.
Pattern note: Like AI Chat, "Submit External Learning" is gated at the tenant level — it's a configuration toggle, not a per-user permission. If end users at a tenant are reporting that they can't find a way to submit external learning, the first thing to check is the tenant setting in Admin Portal → Settings → General.

Tabs

  • Account Info (default tab)
  • Transcripts — the user's full learning transcript / completion history.

Editing your username

The user's Username is the editable display name shown in the Welcome heading on Home (`Welcome, [Username]`) and at the top of the My Account page. Click the pencil icon next to the username on the Account Info tab to open the Edit Username modal — a single text field plus Cancel / Save.

Note on the Welcome heading: the Home page heading reads Welcome, [Username] — always the Username, never a tenant name. The Username is freely editable by the end user, so it can look like a real name, a company name, or anything else.

Account Info sections

  • Commands in Autodesk Products — manage Related Learning Command preferences (which Autodesk products show Pinnacle suggestions in-app).
  • Language — set preferred language. Pinnacle supports 130+ languages with auto-translated asset titles, descriptions, and transcripts.
  • Password — Change Password (only available for users who sign in with a password — i.e., not SSO-only users).
  • Notifications — toggle email notifications. Note: an admin can also disable email notifications tenant-wide.
  • Download Add-Ons — links to download Pinnacle Management Utility and Pinnacle User Tools (MSI installers).

Activate on This Device

After installing Pinnacle User Tools, an "Activate on This Device" button appears in Account Info. Clicking it activates the plugin in the user's installed Autodesk products. Close any open Autodesk applications first.

Administration Portal Overview Admin Portal

Where Home page → left nav → MANAGEMENT section → Administration Portal

The Administration Portal is the admin-side UI for Pinnacle Series. It's where users with admin-tier permissions manage content, users, groups, libraries, settings, and reports.

Accessing the Admin Portal

From the Home page, the MANAGEMENT section appears in the left nav (only if the user has any admin-tier permission). Below the section header, the only item visible to most users is Administration Portal. Click it to enter the admin UI.

Classic UI legacy: Older training material may say "click the cogwheel icon in the top right." That's the Classic UI. In Peak Experience, the path is via the MANAGEMENT section in the left nav.

Admin Portal left navigation (full)

The Admin Portal left nav is a flat list of top-level items. When all admin permissions are granted, the order is:

  1. Admin Home
  2. Assignments ▸ (expandable: Assignment Management, Auto-Assignments)
  3. Content
  4. Libraries
  5. External Learning ▸ (expandable: Records Management, Setup)
  6. Reports
  7. Users ▸ (expandable: Standard Users, External Users)
  8. Groups
  9. Theme Customization
  10. Settings ▸ (expandable: General, Users, Groups) — does not contain Security & Authentication or Integrations
  11. Security & Authentication — top-level (sibling of Settings, not a child)
  12. Integrations — top-level
  13. KnowledgeSmart Admin Area — top-level
Common mistake: Earlier docs and some test cases described Security & Authentication and Integrations as children of Settings. They are top-level items. Settings only contains General, Users, and Groups subpages.

Permission-filtered nav

The left nav only renders the items the user's permissions grant access to. Examples:

User's permissionWhat they see in the left nav

Content Administrator onlyAdmin Home, Content, Libraries
Administrator (User Administration) onlyAdmin Home, Users ▸, Groups, Settings ▸ (subset)
All admin permissionsEvery item in the list above

Top bar

  • User avatar (top-right) → Switch Tenant, Sign Out.
  • Tenant name displayed near the top.

Admin Home

Where Admin Portal → left nav → Admin Home

The default landing page when an admin enters the Administration Portal. Acts as a dashboard / launchpad for the admin's primary tasks.

Required permissions: Any admin-tier permission. The page itself is not gated separately — if you can enter the Admin Portal, you see Admin Home.

Assignments & Auto-Assignments Admin Portal

Where Admin Portal → Assignments ▸ → Assignment Management or Auto-Assignments

Assignment Management

The Assignment Management page is where admins create, monitor, and manage all assigned learning and assessments.

Assign New flow

  1. Click the green Assign New button (top-right of the Assignment Management page).
  2. A menu appears — select Content or Assessment.
  3. The Assign Content / Assign Assessment sub-page opens with a search-and-filter panel for picking what to assign and a user/group picker for who to assign to.
  4. Set due date, training thresholds (if applicable), and exclusion rules, then click Assign.

Breadcrumb on sub-pages: Assignment Management > Assign New.

Common admin actions

  • Filter existing assignments by date range, content type, status.
  • Re-Enroll a user in an assignment.
  • Resend assignment notification email.
  • Set expiration dates for assessments.
  • Exclude users who already completed prior versions.

Auto-Assignments

Auto-Assignments are rules that automatically assign content or assessments based on conditions — typically group membership or assessment results.

Creating an auto-assignment

  1. Click Create New Auto-Assignment.
  2. Select the trigger type: Select Content or Select Assessment.
  3. Pick the target group(s).
  4. If assessment-driven: define training thresholds (e.g., users scoring <30% get full path; users scoring >80% are excluded).
  5. Save and activate.
Required permissions:
  • Assignment Administrator — assign & track for any user/group.
  • Assignor — assign & track for users they manage or groups they own (scoped).

Content Admin Portal

Where Admin Portal → Content

The Content area is where admins author, organize, publish, and manage all custom content. Pinnacle supports several content types, each with its own authoring page.

Content types

  • Pinnacle Document — native authored doc (cheatsheet-style).
  • Video — uploaded or referenced video, with chapters, transcripts, and auto-transcribe.
  • External Document — Word, Excel, PowerPoint, PDF.
  • Quiz — for embedded knowledge checks.
  • Workflow — step-by-step process diagrams (with a visual editor; "Launch Editor" button).
  • Course — a structured collection of content with sections.
  • Learning Path — a sequence of courses.
  • SCORM Course — imported SCORM package.
  • Live Event — live training session (Classic only — not yet in Peak).

Common content actions

  • Add New — opens menu for picking content type.
  • Bulk Modify — select multiple items via checkboxes, then update metadata (keywords, tags, topics) in one go.
  • Search and Replace — global find/replace across content. Supports preview before applying.
  • Move — relocate content between folders.
  • Duplicate — exact copy for reuse or customization.
  • Recycle — temporarily delete (sends to Recycle Bin).
  • Share — generate a shareable link for selected content.

Tools Menu & Recycle Bin

The Tools Menu (kebab icon on the Content page toolbar) provides:

  • Recycle Bin — review and restore deleted content. This is the only entry point to the Recycle Bin.
  • Bulk Actions — Move, Copy, Delete, Bulk Modify icons.
  • KnowledgeSmart Mapping — link content to KS assessment outcomes.
Recycle Bin path: Admin Portal → Content → Tools Menu (kebab) → Recycle Bin. It is not a left-nav item.

Folders

The Content area uses folders to organize content. Common per-folder actions: Create Copy, Share (Add Groups, Add Users), Topics association, Start Modifying (enters bulk-edit mode).

Required permissions (descending access):
  • Content Administrator — full access to all content and metadata, can create/manage libraries.
  • Content Publisher — author, publish, and manage content within libraries shared with them.
  • Content Author — author and manage draft content; can't publish independently.

Libraries Admin Portal

Where Admin Portal → Libraries

The Libraries page is where admins create, manage, and share libraries — the top-level containers that group content for visibility control.

Library types

  • Subscription Libraries — Eagle Point-provided. Visibility can be managed by a Content Administrator.
  • Custom Libraries — created by the customer. Listed under the admin's name (e.g., "Nick Cartony Libraries").
  • Featured Libraries — any library the admin flags as featured. Featured libraries appear in their own section above the regular Libraries section on the end-user Home page.

Common actions

  • Create Library — make a new custom library.
  • Delete — remove a custom library (Content Administrator only).
  • Topics — assign topics/subtopics to organize the library.
  • Share — grant access to specific users or groups.
  • Feature — flag a library as Featured for the tenant.
Required permissions:
  • Content Administrator — full access: create, delete, share, feature.
  • Content Publisher — limited: only views/manages libraries explicitly shared with them.

External Learning Admin Portal

Where Admin Portal → External Learning ▸ → Records Management or Setup

External Learning is the area for tracking training a user completed outside Pinnacle Series — third-party courses, conferences, certifications — so it shows up in the user's record alongside in-platform learning.

Records Management

Where admins view, create, approve, and manage external learning records.

  • Create Record — manual entry: Training Title, Completed Date, Training Provider, Duration, Rating, Feedback, optional file upload.
  • Approve / Deny — for pending external learning requests submitted by users (Assignment Administrator scope).
  • Bulk import — Browse Files → Preview Import → Continue Import → Process Import flow.
  • Filter by user, group, manager, parent/child group.

Setup

Where admins define the catalog of External Learning Activities users can request credit for.

  • Create Activity — Training Title (required), Training Provider, Duration (Minutes).
  • Edit / Delete existing activities.
Required permissions:
  • Assignment Administrator — manage and approve external learning for all users.
  • Assignor — manage and approve only for their managed users / owned groups.

Reports (Peak Reporting) Admin Portal

Where Admin Portal → Reports

Peak Reporting is a unified reporting suite launched in August 2025. It replaces the older Management Utility-based reporting for most use cases.

The five reporting pillars

PillarAnswers questions about
User Management ReportsEngagement, login activity, adoption trends. Who is provisioned? Who is active in Pinnacle and KnowledgeSmart?
Usage ReportsContent access, search behavior, user interest. What are people doing? What resource types are most popular? What are users searching for?
Learning ReportsCourse progression, completions, learning paths. How are learners progressing? How much time are users spending?
Assessment ReportsCompetency, scores, time taken, gaps. What competencies and skill gaps do we have?
Content ReportsInventory, content types, publishers, governance, library structure. What content do we own or need to govern?

Key features

  • Saved Views — admins can save filtered report configurations for reuse.
  • Scheduled Report Alerts — automatic email alerts when a report meets a defined threshold.
  • Targeted permissions — Reporting Administrator (sees all users) vs. Reporting Manager (sees only managed users).
  • Power BI export — Usage Reports support export for use in Power BI dashboards.
Required permissions:
  • Reporting Administrator — full reporting access for all users.
  • Reporting Manager — scoped to users the person manages or groups they own. Includes non-user-specific Content data.
Naming note: The "Reporting Administrator" permission was previously called "View Reports." Older training material may still use the old name. The "Reporting Manager" scoped role is new and didn't exist under the old naming.

Users Admin Portal

Where Admin Portal → Users ▸ → Standard Users or External Users

The Users dropdown in the Admin Portal expands to two sub-pages: Standard Users (your organization's users) and External Users (vendors, contractors, project partners). Groups is a separate top-level item — not nested under Users.

Standard Users page

Lists all users in the tenant. Filterable by domain, group, manager.

Adding users

  • Add User button → opens a menu with Manual and Import from Spreadsheet options.
  • Manual: enter Full Name (required), Email (required), then Add User.
  • Import from Spreadsheet: Browse Files (one file at a time) → Import Options (Add new / Update existing / Delete missing / Use first row as headers / Create groups if missing) → Map fields (Display Name and Email are required) → Preview Import → Process Import.

Bulk edit

Select users via checkboxes → Edit Users menu offers:

  • Exclude Users from SSO
  • Exclude Users from Active Directory Sync
  • Exclude Users from Synchronized Groups
  • Users Must Change Password at Next Sign-in

Per-user actions

  • View Full Profile — opens User Details page with Edit Profile button.
  • Edit User — modify user details, permissions, group membership.
  • Delete User — confirmation prompt, then soft-delete.
  • Send Sign-in Credentials — emails the user a sign-in link with a customizable message.

Common user attributes

  • Full Name, Email
  • Group membership (one or many)
  • Manager (drives Reporting Manager / Assignor scope)
  • Permissions (set per user OR inherited from groups via the Inherit Permissions toggle)
  • Exclude from SSO toggle (under User Settings)
  • Product Expertise / Areas of Interest

External Users page

External Users is a separate user type for people outside your organization who need scoped access to your Pinnacle Series tenant — typically vendors, contractors, or project partners.

  • External Users have their own page (sub-item under Users dropdown).
  • Lifecycle: Invite → Activate → Assign content → Expire on demand.
  • Same person can be an External User in multiple organizations.
  • Their activity does show up in your internal reports.
  • External Users do not count against your standard user license.

SCORM Export Admin Portal New feature

Where Admin Portal → Content / Learning Paths → Export as SCORM (on eligible learning paths or courses)

SCORM Export lets a customer package selected Pinnacle Series learning content into SCORM files for upload to their own LMS (Cornerstone, Workday Learning, a corporate LMS, etc.). Learners launch the activity from inside that external LMS, but their progress, completions, and scores are reported back to Pinnacle Series.

This is new functionality. Detailed UI flow and test coverage are still being developed. Behavior captured here may evolve. Check Pinnacle_UI_Facts.md §6a for the most current capture.

What can be exported

  • Learning Paths — only a limited subset of learning paths is eligible for SCORM export. Not every learning path will have the option.
  • Courses within a learning path — when exporting, the admin can choose to export the entire learning path OR specific course(s) from within it.

How it works for the learner

  1. An admin exports an eligible learning path (or selected courses) as a SCORM package from Pinnacle Series.
  2. The customer uploads the SCORM package into their own LMS.
  3. A learner launches the SCORM activity from inside that LMS — they do not sign in to Pinnacle Series directly.
  4. The learner consumes the content. Progress and completion are reported back to Pinnacle Series via the SCORM package's link to the issuing tenant.

Prerequisites for the learner to consume the content

All four conditions must be true at the moment the learner launches the SCORM activity, not just at export time:

  1. The learner has an active Pinnacle Series account in the same tenant the SCORM package was exported from.
  2. The learner's email in the external LMS matches their email in Pinnacle Series — email is the identity correlation key.
  3. The learner's Pinnacle Series permissions and content visibility still allow access to the underlying content (libraries shared with their group, content not deleted, etc.).
  4. The tenant's Pinnacle Series subscription is active.

What blocks access

If any of the conditions above fails, the SCORM activity will not launch successfully or will not report progress back. Common blocking conditions:

  • Pinnacle Series subscription has expired.
  • The learner does not have a Pinnacle Series account in the tenant the package was exported from. (E.g., a vendor or external partner who is not in the tenant cannot consume the package.)
  • The learner's email in the external LMS does not match their email in Pinnacle Series.
  • The underlying content is no longer accessible to the learner in Pinnacle Series — library not shared with their group, content unpublished, or content deleted.
What CSM / Support should ask first when learners report broken SCORM activities:
  1. Is the learner's email the same in both the external LMS and Pinnacle Series?
  2. Is the learner still active in Pinnacle Series in the tenant the package was exported from?
  3. Does the learner still have access to that specific content in Pinnacle Series (library sharing intact, content still published)?
  4. Is the tenant's subscription still active?
Common Sales question: "Can a vendor or external partner who isn't in our Pinnacle tenant consume our SCORM package?" Answer: No. The package is bound to the issuing tenant. Each learner consuming the SCORM package must have an active account in that same tenant.
Required permission to export SCORM: Likely Content Administrator or Content Publisher, but this has not been visually confirmed yet. Will be updated as test cases are authored.

Common Questions

What permissions does someone need to add users to Pinnacle Series?

Administrator (under User Administration). This grants the ability to add users manually, import them via spreadsheet, sync them from Entra ID / Active Directory, and assign them to groups. See Users and Permissions Reference.

What permissions are needed to publish content?

Content Administrator (full) or Content Publisher (limited to libraries shared with the publisher). A Content Author can create content but cannot publish it independently.

What permissions are needed to view reports?

Reporting Administrator for all users, or Reporting Manager for users the person manages or groups they own. (Older docs may call this "View Reports" — that name has been renamed.)

How do I get to the Recycle Bin?

Admin Portal → Content → Tools Menu (kebab icon on the toolbar) → Recycle Bin. It is not a left-nav item.

Where do I configure SSO?

Admin Portal → Security & Authentication (top-level item, not under Settings). Then scroll to the Single Sign-On Authentication section.

Why doesn't a user see SPARK?

SPARK is enabled per group. If the user's group does not have SPARK toggled on, they won't see SPARK in the left nav or on the Home page. Check Admin Portal → Groups → select the group → Edit Group → SPARK toggle. Also note: if no relevant libraries are shared with the group, the SPARK feed may appear empty.

Why does an Admin Portal user see fewer nav items than expected?

The Admin Portal left nav is permission-filtered. Each user only sees the sections their permissions grant access to. If a user has only Content Administrator, they'll see Admin Home, Content, and Libraries — and nothing else. Granting additional permissions (e.g., Administrator for Users, Assignment Administrator for Assignments) expands what they see.

Why is "Naviate" spelled that way?

Naviate is the name of a specific customer (a Symetri subsidiary) for whom Eagle Point built a custom SSO integration. It is not a typo of "Navigate." See Sign In & Authentication for the Naviate auth flow.

What's the difference between Peak and Classic?

Peak Experience is the modern UI; Classic is the legacy UI. Tenants migrate at their own pace. Peak is enabled by two independent tenant toggles: Peak Experience Configuration (admin UI) and Peak Experience for End Users (Home page). See Peak vs Classic.

Does AI Chat use customer data to train the model?

No. Microsoft Azure OpenAI does not use customer data to train or improve the model. Inputs and outputs are not retained. AI Chat uses RAG to pull from content the user has explicit access to. See AI Chat.

Where do users download Pinnacle User Tools?

Top-right user avatar → Profile & Settings → Account Info tab → Download Add-Ons section. The page lists both Pinnacle Management Utility and Pinnacle User Tools. Visibility of these download links is controlled by the Administrator permission on the tenant.
